addBPM¶

class primitives_GMOS.GMOSPrimitives[source]¶

This is the class of all primitives for the GMOS level of the type hierarchy tree. It inherits all the primitives to the level above , ‘GEMINIPrimitives’.

addBPM(rc)[source]¶

This primitive is used by the general addDQ primitive of primitives_GEMINI to add the appropriate BPM (Bad Pixel Mask) to the inputs. This function will add the BPM as frames matching that of the SCI frames and ensure the BPM’s data array is the same size as that of the SCI data array. If the SCI array is larger (say SCI’s were overscan trimmed, but BPMs were not), the BPMs will have their arrays padded with zero’s to match the sizes and use the data_section descriptor on the SCI data arrays to ensure the match is a correct fit.

Using this approach, rather than appending the BPM in the addDQ allows for specialized BPM processing to be done in the instrument specific primitive sets where it belongs.

Parameters:
  • suffix (string) – Value to be post pended onto each input name(s) to create the output name(s).
  • logLevel (integer from 0-6, 0=nothing to screen, 6=everything to screen. OR the message level as a string (ie. ‘critical’ , ‘status’, ‘fullinfo’...)) – Verbosity setting for log messages to the screen.
